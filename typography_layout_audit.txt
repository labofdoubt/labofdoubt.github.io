### Precedence / what “wins”
- **Global CSS** (`astro/src/styles/global.css`) applies site-wide via `BaseLayout.astro`.
- **Inline styles win over global CSS**: the post body wrapper uses `style="... text-align: left"`, which overrides `main { text-align: center; }` for the rendered Markdown content in posts.


### Current behavior summary
- **Text column max width**: ~**740px** (`--contentw`), centered with `padding: 3rem 1.25rem`.
- **Font sizing**: mostly **non-responsive** (browser default base font size), with the exception of the home page title (`.home-title` uses `clamp`).
- **Images/figures**: default figures are capped to **620px**; `.wide` figures are allowed up to **min(740px, 100vw - 2.5rem)** and centered.
- **Overflow handling**: **no explicit overflow rules** for code/tables/equations at the CSS layer right now.
